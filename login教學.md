# 📅 登入活動配置操作手冊：活動條件設定

## 手冊概述
本章節主要說明在活動發布前的核心邏輯設定，包含審核方式的選擇以及獎勵階梯的詳細配置。請依照以下說明進行操作，以確保活動運行的準確性與安全性。

---

## 一、 審核方式配置區塊
此區塊用於定義用戶參與活動或領取獎勵時的審核邏輯，確保資金發放受控。

* **【功能說明】**：本區域提供活動參與資質的過濾機制。管理員可決定由系統自動判別或投入人工資源進行逐案複審，並可針對特定金額設定審核門檻，平衡運營效率與資金安全。
* **【操作細節】**：
  * **人工審核**：選中後，所有符合條件的申請均需經後台管理員手動核准後方可發放獎勵。
  * **自動審核**：選中後，系統將根據預設規則自動觸發獎勵發放，無需人工干預。
  * **金額門檻設定 (CNY)**：在輸入框中填寫具體金額。若設定該值，當獎勵金額超過此數字時，系統將強制轉為人工審核。
    > **注意**：填寫 `0` 或留空可能代表所有金額均需經過手動審核（具體視系統底層邏輯而定，建議初次設定時進行測試）。

---

## 二、 獎勵設定表
此區塊用於詳細規劃活動的獎項層級、中獎機率及發放數量。

* **【功能說明】**：透過表格化管理，精確控制活動的階梯獎勵結構。管理員可定義不同名次的獲獎價值、名額限制以及抽獎機率，實現對活動成本（ROI）的精準控管。
* **【操作細節】**：管理員需針對表格中的每一行（名次）完成以下維度設定：

| 欄位名稱 | 操作說明 |
| :--- | :--- |
| **獎項 (輸入金額)** | 請輸入該名次對應的獎勵價值（如：現金金額或點數）。 |
| **中獎人數** | 設定該獎項總計可獲獎的人數上限。 |
| **剩餘數量** | 顯示目前庫存剩餘可發放的名額（系統自動帶入或手動初始化設定）。 |
| **概率設置** | 輸入該獎項的中獎機率（建議以小數表示，如 `0.1` 代表 `10%`）。請確保所有獎項機率總和符合活動邏輯。 |
| **打碼限制** | 設定獲獎者領取獎勵後需達到的流水（打碼）要求。填寫 `0` 表示不設限制，領取後即可提現。 |

---

## 三、 注意事項
* **保存進度**：在切換至「風控設定」或「領取條件」前，請務必點擊頁面右上角的「下一步」以確保當前配置已成功暫存。
* **機率校對**：調整獎勵設定表後，請務必重新計算概率總和，避免因機率設定錯誤導致獎項發放異常。
* **預覽檢查**：所有配置完成後，建議在最後的「送出預覽」步驟中再次核對金額與人數限制。

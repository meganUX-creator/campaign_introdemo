# 優惠碼新增流程與設置說明

這份介面截圖顯示的是一個後台系統的「優惠碼設置」模組。該頁面主要用於定義優惠活動的基礎資訊、時間、幣種、審核流程以及核心的參與規則與優惠券配置。

以下是根據圖中標注區域（1-4 及 a.1-a.3）的詳細說明：

## 一、 基礎配置區域 (1-4)

| 編號 | 模組名稱 | 功能說明 |
| :--- | :--- | :--- |
| **1** | **前台展示名稱** | 設定該優惠活動在用戶端（App/網頁）顯示的標題。這是吸引用戶點擊的第一眼資訊。 |
| **2** | **時間設定** | 定義活動的有效期限。可選擇「指定時間段」（需選擇開始與結束日期）或「長期展示」。 |
| **3** | **幣種設置** | 選擇該優惠碼適用的法定貨幣或虛擬貨幣種類（如：CNY, USD, USDT 等）。 |
| **4** | **審核方式** | 設定領取優惠後是否需要人工介入。可選「人工審核」以提高風控安全性，或「自動審核」以提升用戶體驗。 |

## 二、 優惠內容與參與規則 (5)

此區域為整個設定的核心，定義了用戶獲得優惠券必須達成的門檻。

### a.1 參與條件一：基礎帳號與行為門檻
此區塊主要針對帳號的安全性與社交屬性進行過濾：
* **個人信息**：可勾選必須綁定的聯絡方式（如：電子郵件、WhatsApp、Telegram、QQ、Zalo）。
* **支付條件**：規定用戶必須綁定特定的支付工具（如：電子錢包、銀行卡、虛擬幣錢包）。
* **手機驗證**：強制要求手機號碼綁定狀態。
* **邀請/登錄行為**：設定需「邀請好友數」或「連續登錄天數」達標。

### a.2 參與條件二：資金行為門檻
此區塊針對用戶的資金貢獻度進行設定：
* **存款要求**：檢查用戶的有效存款次數或金額。
* **投注要求**：檢查用戶的有效投注額或流水是否達標。

### a.3 新增優惠碼清單
這是最終產出優惠券的地方，支援批量生成與管理：
* **類型切換**：可選擇「系統生成」隨機碼或「自定義」特定字串的優惠碼。
* **核心參數**：
    * **金額**：單張優惠券的價值。
    * **數量**：該組優惠碼的總發放張數。
    * **每日限制**：單個帳號每天最多可使用的次數。
* **操作**：點擊右側「新增優惠碼」按鈕可增加多組不同面額或規則的代碼。
